\# HAR 数据采集协议 v1.0



\## 1. 实验目的

采集高质量、标注清晰的 IMU 数据，用于训练能够区分 8 种人体活动的分类器。重点解决“数据泄露”与“类别不平衡”问题。



\## 2. 硬件配置

\- \*\*主控\*\*：ESP32-S3-N16R8

\- \*\*传感器\*\*：MPU6050 (加速度计+陀螺仪) + HMC5883L (磁力计)

\- \*\*佩戴位置\*\*：\*\*腰部右侧\*\*（皮带处）或 \*\*惯用手手腕\*\*。

&#x20;   - \*注意：全员必须统一佩戴位置，严禁混用位置数据。\*

\- \*\*采样率设置\*\*：

&#x20;   - 加速度/陀螺仪：\*\*50 Hz\*\* (满足奈奎斯特采样定理，且降低存储压力)。

&#x20;   - 磁力计：10-20 Hz (可选，视 I2C 带宽而定)。



\## 3. 被试方案 (Subject Protocol)

为避免模型只认识“某一个人”的动作（过拟合），必须遵循\*\*被试独立原则\*\*。



\- \*\*被试人数\*\*：至少 \*\*3名\*\* 不同体型/性别的志愿者。

\- \*\*采集量要求\*\*：

&#x20;   - 每位被试对 \*\*每一类动作\*\* 至少采集 \*\*3次\*\*。

&#x20;   - 每次持续时长 \*\*≥ 60秒\*\* (保证有足够的数据窗口进行切分)。

\- \*\*文件命名规范\*\*：`Sub\[ID]\_Act\[Action]\_Trial\[TrialNum].csv`

&#x20;   - 例如：`Sub01\_Walk\_Trial01.csv`



\## 4. 动作执行规范

| 类别 | 执行说明 | 注意事项 |

| :--- | :--- | :--- |

| \*\*静坐/站立\*\* | 保持自然姿态，不要刻意僵硬，允许微小晃动。 | 需记录开始和结束时的静止状态以便对齐。 |

| \*\*平地步行/跑步\*\* | 保持匀速，沿直线行走至少 20米往返。 | 避免急停急转。 |

| \*\*上楼/下楼\*\* | 正常步伐，手扶或不扶栏杆需全程保持一致。 | \*\*高风险混淆项\*\*，务必保证楼层高度一致。 |

| \*\*跌倒\*\* | \*\*(注意安全!)\*\* 在软垫上进行模拟跌倒（向前/向后/侧向）。 | 必须有保护人员在场！ |



\## 5. 标签与预处理规则 (关键！)

为了防止模型学习到错误的过渡特征，必须执行\*\*过渡段裁剪\*\*：



1\.  \*\*打标方式\*\*：采用“动作稳定后开始计时，动作结束前停止计时”。

2\.  \*\*过渡段切除\*\*：

&#x20;   - 每个样本的 \*\*前 2秒\*\* 和 \*\*后 2秒\*\* 视为“动作切换期”（如从站起到迈步的过程）。

&#x20;   - \*\*处理操作\*\*：在后续数据处理脚本中，必须将这前后各 2秒的数据\*\*丢弃\*\*，只保留中间稳定的 56秒数据。

3\.  \*\*异常值标记\*\*：如果在采集中途出现打喷嚏、接电话等干扰，需在文件名后备注 `\_bad`，后期予以剔除。



\## 6. 数据集划分原则 (防泄露铁律)

\- \*\*绝对禁止\*\*：将同一个人的数据拆分到训练集和测试集。

\- \*\*正确做法 (LOSO/Subject-Independent)\*\*：

&#x20;   - \*\*训练集\*\*：被试 A + 被试 B 的所有数据。

&#x20;   - \*\*测试集\*\*：被试 C 的所有数据。

&#x20;   - \*验证集\*：从训练集中再分出 20% 用于调参。
