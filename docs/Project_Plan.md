\# 可穿戴人体活动识别系统 (HAR) - 项目计划书



\## 1. 项目概述

本项目旨在基于 ESP32-S3-N16R8 微控制器与 MPU6050/HMC5883L 传感器，构建一套低功耗、高精度的可穿戴人体活动识别系统。系统将实时采集运动数据，通过边缘计算自动区分日常活动与异常状态，并最终在嵌入式端实现实时演示。



\## 2. 核心目标与类别定义

根据产业需求与算法难度，我们将识别任务划分为两个层级：



\### 2.1 基础必做组（6类）

\*   \*\*静坐 (Sitting)\*\*

\*   \*\*站立 (Standing)\*\*

\*   \*\*平地步行 (Walking)\*\*

\*   \*\*跑步 (Running)\*\*

\*   \*\*上楼 (Upstairs)\*\*

\*   \*\*下楼 (Downstairs)\*\*

> \*\*目标：\*\* 保证高准确率，作为系统的核心功能基线。



\### 2.2 进阶挑战组（2类）

\*   \*\*跌倒 (Falling)\*\*

\*   \*\*异常静止 (Abnormal Static/Post-fall)\*\*

> \*\*目标：\*\* 针对老人监护场景，重点优化“跌倒漏检率”，引入代价敏感学习策略。



\## 3. 技术路线与分工

\- \*\*硬件层 (S/PCB)\*\*：

&#x20;   - 完成 MPU6050 + HMC5883L 原理图设计与 ERC 检查。

&#x20;   - 解决 I2C 总线冲突（使用 GPIO8/9），设计电源管理与去耦电路。

&#x20;   - 负责 PCB 打样与焊接调试。

\- \*\*数据层 (B)\*\*：

&#x20;   - 制定严格的数据采集协议（采样率 50Hz+，窗口化切片）。

&#x20;   - 执行被试独立划分（Subject-independent split），杜绝数据泄露。

\- \*\*算法层 (A)\*\*：

&#x20;   - 特征工程：提取时域（均值、方差）与频域特征。

&#x20;   - 模型训练：对比 SVM、Random Forest 或轻量级 CNN/LSTM。

&#x20;   - \*\*难点攻关\*\*：针对“上下楼”易混淆问题引入重力分量分析；针对“跌倒”设计非对称代价矩阵。



\## 4. 进度安排 (Milestones)

| 阶段 | 时间节点 | 关键交付物 |

| :--- | :--- | :--- |

\## 5. 风险评估

\- \*\*数据质量风险\*\*：若采集动作不标准，将导致模型过拟合。 -> \*对策：严格执行采集协议，录制参考视频。\*

\- \*\*硬件干扰风险\*\*：I2C 通信不稳定。 -> \*对策：预留上拉电阻焊盘，软件增加重试机制。\*
